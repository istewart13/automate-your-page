Important Concepts

Lesson 1: Introduction to "Serious" Programming

Computer

A computer is a universal machine that can be programmed to perform any computation, and they can perform billions of operations per second. In order to do this we need to write a program that gives specific instructions to the computer that it is capable of understanding. A program is a precise sequence of steps that a computer can follow to do something useful. Programs need to be written in a programming language, Pythonis just one example of this.

Python

When you run a Python program, an interpreter converts the code that you wrote into something that the computer can understand and execute. High-level language instructions that are easy for humans to read/write, are therefore turned into machine code. An expression in Python is any legal statement. Very minor errors can cause the code not to work. For example print 7 is legal, but  print 7 (with any number of spaces before it) or Print 7is not.

Grammar

Programming languages have grammars just like natural languages do. A grammar is a specification of what is "correct" and what is "incorrect." Peopleare often capable of interpreting statemnts where the grammar is a little bit off, but computers can't. This means that we have to write code that is fully correct, otherwise we will generate a syntax error and our code won't run.

Lesson 2: Variables and Strings

What is a variable?

Variables provide a way for programmers to be able to store values in a labelled 'box'. They should be used to improve the readibility of code.For example: my_name = "Iain"

Assignment statements

Assignment statements allow us to assign a particular value to a variable. In this case the use of the equals sign is not testing for equality (use "==" to do this), but is storing the given value in the variable given. For example: days_in_week = 7

Why are variables useful?

Variables can be useful to programmers in many ways:
  • They improve code readability by using names that make sense to humans.
  • They give us a way to store values which can be used again later in the code.
  • They give us a way to update the value of something, which can be used in conjunction with other key programming concepts (eg. to iterate through the values in a loop).

Indexing strings

Indexing strings is very resiliant and works even if we index positions that don't exist in the string. eg. t = "Hi" print t[:3] + t[3:] returns Hi It is different if we try to index particular characters by themselves though. eg. t[3] returns a string index out of range error

Find method

The find method lets us search for a substring within the string on which it is called. The output is a number that gives the first position in the search string where the target string occurs (or -1 if it is not found). You can also add a starting location from which to begin searching the string as a second parameter. eg. "namename".find("n", 3)

Lesson 3: Input -> Functions -> Output

Functions

  • Functions take input, do something with it, and then produce output.
  • Functions are created by starting a line of code with the keyword def and then giving a function name followed by the function parameters in parentheses. These parameters will be replaced by actual values when the function is called.
  • The return keyword tells Python exactly what the function should produce as output
  • In the 'body' of the function, we write the code that specifies what to do with the input parameters. For example the following code could be the definition of a function called square:
    def square(x):
        answer = x * x
        return answer
  • To use a function, we write the name of the function followed by the value(s) we want to give it in parentheses. Like this:
    print square(6):
    >>>36
  • Functions significantly improve the reusability of a piece of code - they can be created once and the called any number of times.

Lesson 4: Control Flow & Loops: If and While

if statements

If statements are used along with comparisons to make decisions in our code, different code will be executed depending on the result of the comparison.
if <TestExpression>:
    <Block>
eg.
x = 5
if x > 4:
    return "x is greater than 4"
You can also use the or operator in conjunction with if statements. In this case if either of the expressions is true, then the value is True.

while loops

In a while loop, the block can execute any number of times as long as the test expression remains true.
while <TestExpression>:
    <Block>
eg. To print the numbers 0 to 9:
i = 0
while i < 10:
    print i
    i = i + 1
Break gives us a way to stop the loop even whilst the test condition is true. Its format is:
while <TestExpression>:
    <Code>
    if <BreakTest>:
        break

Equality comparisons

Python provides lots of different operators for doing comparisons. The output of a comparison is a Boolean value (either true or false).
Operation Meaning
< strictly less than
<= less than or equal
> strictly greater than
>= greater than or equal
== equal
!= not equal

Lesson 5: Debugging

Debugging strategies

  1. Examine error messages when programs crash: The last line of Python Tracebacks will tell you what went wrong. Reading backwards from there will tell you more about where the problem occurred.
  2. Work from example code: If your modified code doesn't work, comment it out and do step-by-step modifications to the example code until it does what you want.
  3. Make sure examples work: Just because you find example code doesn't mean it will work in your system. Check the example code you're using to make sure it behaves the way you expect.
  4. Check (print) intermediate results: When your code doesn't crash, but doesn't behave as expected, add print statements to your program to see where in the code things stop behaving correctly.
  5. Keep and compare old versions: When you have a working version of your code, save it before you add to the code. This will give you something to go back to if you introduce too many new bugs.

Lesson 6: Structured Data: Lists & For Loops

Creating and accessing lists

A string is a sequence of characters, whilst a list is a sequence of any structured data (characters, strings, numbers, other lists etc). We use square brackets [] to identify a list in python, and elements are separated by commas. Lists can contain a mixture of strings and numbers if required.
eg. name = ['I','a','i','n']
Lists can be 'accessed' and 'sliced' in the same way that strings can.
eg. print name[1] --> a
eg. print name[1:3] --> ['a','i']

Nested lists

List can contain other lists within them.
eg. nested_list = ['I',['a','i'],['n']]
You can access particular values which are contained in a list within a list as follows:
third_letter = nested_list[1][1]

Mutation and aliasing

Lists support mutation, whilst strings do not. Mutation means that we can change the value of a list after it has been created.
eg. name = ['I','a','i','m']
name[3] = 'n'
print name --> ['I','a','i','n']
If there are two or more pointers to a list and the list is changed, then all pointers will now point to the updated list. Need to be careful with this, as changing one variable could result in another being updated unintentionally.

Aliasing is when there are two (or more) different ways to refer to the same object.

List operations

Append method is used to add an element to the end of a list. Its format is
<list>.append(<element>)

Addition method behaves like the concatenation operation for strings, and produces a new list which contains all of the values being combined.
<list>+<list>
eg.[0,1] + [2,3] --> [0,1,2,3]

Len works for any object that is a collection of things, the output is the number of elements in the input. It only counts the number of out elements however, so a list within another list only counts as one element.
len(<list>)
eg.len([0,1,2,3]) --> 4
eg.len("Udacity") --> 7

For loops


for <name> in <list>
    <Block>
eg. for <name> in <name_list>
    <print name>

This can be used in conjunction with a range function which generates a list of values in the given range (upto but not including the final value)
range([start], stop[, step])
eg. print range(5) -->
1
2
3
4
eg. print range(3,6) -->
3
4
5
eg. print range(4,10,2) -->
4
6
8

Index method and in operator

You pass in a value, and the output of the method is the first position at which the value exists in the list (or an error if the value isn't in the list).
<list>.index(<value>)
eg. name_list.index("Iain")

With the in operator, if the value given is in the list then output is True, otherwise it is false. You can also use the not in operator, which would return the opposite result.
<value> in <list>
eg. 5 in [3,4,5] --> True

Lesson 7: How To Solve Problems

Problem solving process

  1. Make sure that you understand the problem
  2. Work out what the inputs are
  3. Work out what the outputs should be
  4. Understand the relationship between inputs and outputs - based on some examples
  5. Consider how the problem could be solved systematically by a human (write this out in pseudocode)
  6. Develop a simple mechanical algorithm (Break the problem into the smallest component pieces)
  7. Develop the solution incrementally and test each step as you go
